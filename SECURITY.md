# Security Policy

## Supported versions

Security fixes are only guaranteed for the latest state of the default branch and the latest tagged release.

| Version         | Supported |
| --------------- | --------- |
| Latest          | Yes       |
| Older snapshots | No        |

## Reporting a vulnerability

Please do not open a public issue for security-sensitive problems.

Instead:

1. Use GitHub Security Advisories for this repository if private reporting is enabled.
2. If private reporting is not available, contact the repository owner through their listed contact method.

Include:

- a short description of the issue
- steps to reproduce it
- impact assessment
- any suggested mitigation if you already have one

You should receive an acknowledgement as soon as the maintainers are able to review the report.
